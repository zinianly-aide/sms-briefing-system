# OpenClaw 多 Subagent 协作最佳实践

## 目标
在长期项目开发中，由主 Agent 负责主导、拆解、验收，由多个 Subagent 并行执行前端、后端、环境配置等任务。

## 推荐分工
- Chief Agent
  - 拆解架构与任务
  - 定优先级
  - 做主控汇总和验收
  - 对外同步进度
- Frontend-Agent
  - 负责 UI、布局、页面壳、交互实现
- Backend-Agent
  - 负责服务端结构、API、数据模型、业务逻辑
- DevOps-Agent
  - 负责本地环境、运行说明、配置文件、连续性文档

## 协作原则
1. 小步执行，不用超长命令链
2. 任务边界明确，避免多个 agent 改同一块
3. 每一轮执行都要输出：改了什么、commit 是什么、下一步建议
4. 项目决策写入仓库文档，不写人格文件
5. 待办统一写到 TODO.md
6. 并行后由主 Agent 汇总并做交叉 review

## 适用场景
- 长期项目开发
- 多模块并行实现
- 需要阶段性提交与持续汇报

## 当前项目的落地方式
- 文档先行，工程并行推进
- DevOps 先铺环境与连续性
- Frontend / Backend 再分模块逐步落地
